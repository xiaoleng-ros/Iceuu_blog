# Phase 3 开发计划：后台管理系统核心功能

本阶段将完成后台管理系统的核心功能，实现“一次开发、永久无代码发布”的目标。重点在于富文本编辑器的深度集成（自动上传图片）和内容管理流程。

## 1. 基础设施与 UI 组件
- **基础组件封装**: 创建 `Button`, `Input`, `Label`, `Card` 等基础组件，确保后台风格统一（基于 Tailwind CSS）。
- **通用布局优化**: 完善 `Sidebar` 导航菜单，高亮当前路由。

## 2. 富文本编辑器集成 (Core)
- **组件开发**: 创建 `components/admin/Editor.tsx`，使用 `react-quill-new`。
- **SSR 兼容**: 使用 `next/dynamic` 禁用 SSR，防止 Hydration Mismatch。
- **图片上传 Hook**: 自定义 Quill 的 Image Handler，拦截图片上传操作 -> 调用 `/api/upload` -> 插入 CDN 链接。

## 3. 博客内容管理
- **列表页 (`/admin/blogs`)**:
  - 表格展示：标题、状态（草稿/发布）、分类、发布时间、阅读量。
  - 操作：编辑、删除、预览。
- **编辑/发布页 (`/admin/blogs/[id]`)**:
  - 元数据表单：标题、Slug（URL 别名）、摘要、封面图（支持上传）、分类、标签。
  - 编辑器集成：正文编辑。
  - 自动保存/发布逻辑。

## 4. 媒体库管理 (`/admin/media`)
- **图片网格**: 展示所有已上传图片（分页）。
- **上传功能**: 拖拽上传区域。
- **便捷操作**: 点击复制 Markdown/HTML 链接。

## 5. 站点设置 (`/admin/settings`)
- **配置表单**: 网站标题、描述、SEO 关键词、页脚信息、社交链接。
- **数据存储**: 存入 `site_config` 表（Key-Value 结构）。

## 执行顺序
1.  创建基础 UI 组件。
2.  开发 Editor 组件（含图片上传逻辑）。
3.  开发博客列表与编辑页。
4.  开发媒体库与设置页。
5.  更新 Sidebar 链接。
