# Phase 5 开发计划：部署交付与文档编写

本阶段是项目的收尾阶段，重点在于确保项目能够被用户顺利部署和使用。我们将整理环境变量、编写详细的部署文档，并进行最终的清理工作。

## 1. 环境变量标准化

* **检查**: 遍历代码库，确认所有使用的 `process.env` 变量。

* **整理**: 更新 `.env.example`，添加详细注释，区分必填项与选填项。

* **验证**: 确保本地开发环境与生产环境配置的一致性。

## 2. 部署文档 (DEPLOY.md)

编写一份详尽的 Markdown 文档，包含以下章节：

* **前置准备**: 需要的账号（GitHub, Supabase, Vercel）。

* **数据库初始化**: 引用 `supabase_schema.sql`，指导用户如何创建表和 RLS 策略。

* **环境变量配置**: 逐项解释每个 KEY 的作用及获取方式。

* **GitHub 存储配置**: 如何创建 Token 并设置仓库权限。

* **Vercel 一键部署**: 指导连接 GitHub 仓库并填写环境变量。

## 3. 使用手册 (README.md)

更新项目根目录的 `README.md`：

* **项目简介**: 技术栈与核心功能。

* **快速开始**: 本地运行指南。

* **功能特性**: 列出已实现的功能（后台管理、Markdown 渲染、CDN 加速等）。

## 4. 最终清理

* 删除临时的测试脚本（如之前创建的 `scripts/verify-github-config.ts`，已删除但需再次确认）。

* 检查是否存在未使用的依赖或组件。

## 执行步骤

1. 更新 `.env.example`。
2. 编写 `DEPLOY.md`。
3. 更新 `README.md`。
4. 执行全项目 lint 检查（可选）。
